## 🧠 记忆类型分类（认知科学视角）

### 1. **情景记忆 (Episodic Memory)**
- **定义：** 特定事件和交互的记录（时间戳 + 上下文）
- **特点：** 个性化、时间相关、情境化
- **存储：** 向量数据库 / 事件日志
- **示例：** "用户在2025-03-15询问过网络配置问题"

### 2. **语义记忆 (Semantic Memory)**
- **定义：** 事实、概念、规则的知识库
- **特点：** 抽象化、无时间性、通用性
- **存储：** 知识图谱 / 向量数据库
- **示例：** "HTTP 代理地址是 127.0.0.1:7897"

### 3. **程序记忆 (Procedural Memory)**
- **定义：** 技能、工作流、执行模式
- **特点：** 行为导向、可执行、可训练
- **存储：** 规则引擎 / 脚本 / 本能
- **示例：** "访问外部 API 时使用代理"

### 4. **工作记忆 (Working Memory)**
- **定义：** 当前活跃上下文（会话级）
- **特点：** 容量有限、临时性、快速访问
- **存储：** 内存缓冲区
- **示例：** 当前对话的最近 N 轮

### 5. **联想记忆 (Associative Memory)**
- **定义：** 概念间的关联关系
- **特点：** 图结构、语义链接、推理支持
- **存储：** 知识图谱边
- **示例：** "Brave Search → depends_on → Proxy Config"

## 🏗️ 主流系统架构对比

### **Mem0: 向量 + 图谱混合**

**架构：**
```
对话流
  ↓ 实时提取
记忆片段 (向量嵌入)
  ↓ 合并去重
知识图谱 (Pro版)
  ↓ 检索
语义搜索 + 图遍历
```

**特点：**
- ✅ 动态提取和合并
- ✅ 用户/会话/代理层级
- ✅ 托管服务（生产就绪）
- ⚠️ 图谱功能仅 Pro 版
- ⚠️ 无时序建模

**适用场景：** 生产环境快速部署、用户个性化

---

### **Letta (MemGPT): OS 启发架构**

**架构：**
```
┌─────────────────────────┐
│   Main Context (有限)   │ ← Core Memory (核心记忆)
└───────────┬─────────────┘
            │ 溢出时归档
            ↓
┌─────────────────────────┐
│   Archival Memory       │ ← 长期存储
│   (无限容量)            │
└───────────┬─────────────┘
            │ 工具调用检索
            ↓
┌─────────────────────────┐
│   Recall Memory         │ ← 搜索索引
└─────────────────────────┘
```

**特点：**
- ✅ 自编辑记忆（Agent自主管理）
- ✅ 分层架构（有限→无限）
- ✅ 工具调用委托检索
- ✅ 74% LoCoMo（GPT-4o mini）
- ⚠️ 无结构化知识图谱

**适用场景：** 长对话、复杂推理、自主Agent

---

### **Zep: 时序知识图谱** ⭐

**架构：**
```
对话流 + 业务数据
  ↓ 实时提取
Graphiti 引擎
  ↓ 时序建模
时序知识图谱 (Neo4j)
  ├─ T 时间线（事件时间）
  └─ T' 时间线（事务时间）
  ↓ 检索
层次化子图 + 时序推理
```

**核心创新：**
1. **双时序建模 (Bitemporal)**
   - T 时间线：事件发生的实际时间
   - T' 时间线：数据入库的事务时间
   - 支持时间旅行查询

2. **层次化子图**
   - 全局图（所有知识）
   - 会话图（当前上下文）
   - 用户图（个性化）

3. **实时更新**
   - 增量式图谱构建
   - 事实有效性跟踪
   - 来源可追溯

**特点：**
- ✅ **超越 MemGPT 基准**
- ✅ 理解事实的时间演化
- ✅ 支持复杂时序推理
- ✅ 结合对话 + 结构化数据
- ✅ Neo4j 存储（图原生）

**适用场景：** 长期交互、时序推理、企业级应用

---

### **Hindsight: 4种检索并行**

**架构：**
```
查询
  ├─ 语义检索 (向量)
  ├─ 关键词检索 (BM25)
  ├─ 知识图谱遍历
  └─ 时序推理
  ↓ 并行合并
结果融合
```

**特点：**
- ✅ **超越 Mem0 Pro**
- ✅ 所有检索策略（包括免费版）
- ✅ 深度检索能力
- ⚠️ 相对较新

**适用场景：** 高精度检索、复杂查询

---

### **Continuous Learning v2: 原子化本能**

**架构：**
```
工具调用 (100%钩子捕获)
  ↓ 观察
observations.jsonl
  ↓ /evolve 分析
原子化本能 (YAML)
  ├─ 置信度 0.3-0.9
  ├─ 项目/全局作用域
  └─ 领域标签
  ↓ 聚类进化
技能/命令/代理
```

**特点：**
- ✅ 100% 可靠观察
- ✅ 项目隔离 + 全局提升
- ✅ 渐进式置信度
- ✅ 可进化为可执行产物
- ⚠️ 手动触发 `/evolve`
- ⚠️ 无向量检索

**适用场景：** 行为学习、工作流提取、团队知识共享

---

### **MCP Memory Server: 知识图谱**

**架构：**
```
实体 (name + type + observations)
关系 (from + to + relationType)
  ↓ 图查询
search_nodes / read_graph
```

**特点：**
- ✅ 简单易用
- ✅ 结构化知识表示
- ✅ 支持推理
- ⚠️ 无自动提取
- ⚠️ 无时序建模
- ⚠️ 检索能力有限

**适用场景：** 静态知识库、配置管理
